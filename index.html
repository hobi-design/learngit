Git Fundamentals

A quick reference guide for vibe coders who want to be in control of how their code is managed.

Phase 1

Initialize Repository

Start tracking your project files by creating a new local Git repository.

> git init

Initializes a new, empty local version control environment.

Phase 2

Stage Changes

Git doesn't track changes automatically. You choose exactly what goes into the next snapshot.

> git add -p

Review each change hunk-by-hunk before staging it.

Also common: git add <filename> git status
Phase 3

Create a Commit

A commit is a permanent, named snapshot of your staged changes. It records who, what, and when.

> git commit -m "feat: add login flow"

Creates a snapshot in history. Retrievable forever.

See Best Practices for message conventions.

Phase 4

Push to Remote

A remote is a version of your repository on the internet (e.g. GitHub). Push sends your local commits there.

> git push -u origin main

Sends commits to the remote and sets tracking branch.

First time setup: git remote add origin <url>
